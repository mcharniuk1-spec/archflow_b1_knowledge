# ICP Product Surface Production Release — Context Capsule

## Objective

Release the reviewed ArcFlow ICP website, architecture dashboard, and guarded Jarvis surface to the public Git repository and Vercel production, validate responsive design and interaction quality, synchronize the deployed visual baseline to Figma, and update only the corresponding Notion delivery records.

## Scope

- Public three-block website and seven-layer tower assets.
- Documentation-style architecture dashboard and its generated public data.
- Separate Jarvis review surface and serverless contract with provider execution disabled by default.
- Public architecture, ICP, audience, role, workflow, proof-ledger, and release evidence required to explain the product surface.
- Git commit and push, Vercel production deployment, deployed-route QA, Figma capture sync, and bounded Notion status updates.

## Exclusions

- Generated Graphify output and unrelated historical run folders.
- Private runtime, provider credentials, model execution, durable writeback, or production promotion of an agent action.
- Claims of customer outcomes, savings, reliability, or runtime readiness that are not backed by current evidence.

## Evidence Standard

Use `FACT`, `INTERPRETATION`, `HYPOTHESIS`, and `GAP` where conclusions require judgment. A route is release-ready only after local and deployed responsive checks, public-safety scanning, syntax or contract checks, and an independent read-only review.
