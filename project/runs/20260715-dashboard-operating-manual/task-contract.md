# Dashboard operating manual task contract

## Goal

Transform the public ArchFlow dashboard and Jarvis into explicit developer-facing documentation surfaces for the current local/static product: explain every layer, parameter, workflow, knowledge artifact, role/skill contract, configuration boundary, output, and safe handoff without presenting planned or gated capabilities as live.

## Observable done condition

The dashboard has a documentation/manual route, detailed Knowledge Service and Agent Control flows, explicit configuration guidance, skill-by-skill and role-by-role explanation, safe admin/guest local-preview modes, cross-flow local session context, and downloadable architecture/report packets. Jarvis explains setup inputs, prompt requirements, output shape, and can produce/download a browser-local architecture report. The strategic plan is removed from dashboard navigation and all rendered dashboard documentation remains accurate about local/static limits.

## Authority and boundaries

Authorized: public repository documentation, static dashboard/Jarvis UI, generated public data, browser-local demo state, run artifacts, Git commit, and Git push.

Not authorized or not claimed: authentication, individual guest persistence, real subagent launch, provider execution, private corpus ingestion, real repository file creation from the browser, deployment, external writeback, live database, or hosted runtime activation.

## Evidence state

- Proven now: static dashboard/Jarvis, generated public catalog, browser-local exports, guarded review-packet contract.
- Review: parameter documentation and role-to-skill mappings after static/rendered checks.
- Planned/blocked: auth, per-user durable memory, event feed, real agent runtime, provider calls, database, and external action.

## Task graph

1. UI/manual and accessibility reviewer audits dashboard structure and proposes the smallest documentation architecture.
2. Jarvis workflow reviewer audits prompt/setup/report-download sequence and truth boundaries.
3. Skills/roles reviewer reconciles every public skill with agent responsibilities and identifies missing explicit mappings.
4. Integrator implements public-safe documentation views, local admin/guest preview, shared browser session context, report exports, and docs/manual artifacts.
5. Independent reviewer validates claims, navigation, static checks, generated data, and public safety.

## Roles and stop rules

- Reviewers are read-only and may not edit public files, install tools, call providers, or make external writes.
- Integrator owns only implementation and merge order.
- Stop if the requested feature requires a private source, an authentication secret, real durable guest memory, external execution, or a live provider/runtime assertion.
- Keep the browser-local preview clearly labelled and avoid model/persona names as authority claims.
