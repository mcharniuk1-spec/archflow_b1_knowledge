# Sole Summary Document

## Executive Summary

The June 24 discussion defines ArchFlow as a company workflow that turns messy strategic dialogue into structured execution. The immediate priority is not a public marketing push or a broad framework stack. The immediate priority is to prove the internal operating cycle: sanitize an input, extract context, produce a PRD, divide work by agent role, update the knowledge base, review the output, and only then prepare later research and market-facing actions.

The discussion has two operating positions. The builder/operator position focuses on creating the system: knowledge base, automation, LangGraph routing, agent roles, runtime checks, and durable memory. The market/content position focuses on the external path: audience, positioning, content, website structure, lead generation, and proof of willingness to pay. The best next move is to make these positions collaborate through a controlled proof workflow.

## Key Bullet Points

- Build the company by first building the internal system on itself.
- Convert meeting material into structured summaries, PRDs, task plans, and reusable knowledge.
- Keep the knowledge base as durable memory, while LlamaIndex serves runtime retrieval.
- Use LangGraph for state, routing, review loops, approve/revise/block logic, and traceable execution.
- Use CrewAI for named roles and task responsibilities, wrapped by LangGraph.
- Use LangSmith for sanitized trace observability.
- Keep the dashboard read-only until the proof workflow is repeatable.
- Use market and web research after the first proof cycle, not before it.
- Avoid customer-facing claims until audience and payment tests produce evidence.

## First Proof Workflow

1. Sanitize source packet.
2. Validate boundary and source inventory.
3. Run parallel topic analysis.
4. Merge findings into a structured digest.
5. Produce a PRD.
6. Produce an agent-segmented responsibility matrix.
7. Write a knowledge-base update packet.
8. Run review gate.
9. Publish only public-safe outputs.
10. Generate tasks for the next execution stage.

## Decision

Start E1.2 as a controlled proof run. Do not skip directly to web research, full dashboard control, live vault bridge, or customer claims.
