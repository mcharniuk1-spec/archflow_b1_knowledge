# Primary Operator Knowledge Base

Date: 2026-07-03
Status: E1.6 review-ready

## Role

The primary operator knowledge base stores reviewed operating summaries for the lead execution role:

- current goal and accepted scope;
- decisions and approval boundaries;
- final integration notes;
- deployment, GitHub, Notion, and Telegram closeout status;
- critique summaries and next-step gates.

## Use

1. Start from `project/project-plan.md`.
2. Read the latest run handout under `project/runs/`.
3. Read the latest WikiLLM run note under `wiki/runs/`.
4. Use this folder for role-specific summaries only when the information is useful for future operator decisions.
5. If the note is broadly useful, promote the reviewed summary to `wiki/memory.md`, `wiki/insights.md`, or a dated `wiki/runs/` note instead of duplicating it here.

## Do Not Store

- raw private source text;
- private names or identities;
- private Notion links;
- credentials or token values;
- local machine paths;
- raw transcripts, raw audio, screenshots, or private documents;
- unreviewed market or customer claims as facts.

## Current E1.6 Scope

This folder completes the lead-role side of the E1.6 split. It exists to keep execution ownership clear while preserving WikiLLM as the durable shared memory layer.
