# Self-Supporting Agentic Architecture Review

Date: 2026-07-13

Status: final design baseline; several runtime capabilities remain gated.

## Executive conclusion

Loop Engineering was already integrated, but only as an L1 report-first contract. Goal Engineering was not a first-class layer before this review. The architecture now has an explicit goal schema, a clearer layer order, a parameterized architecture-factory skill, Terra/Luna coordination, role packs, adoption gates, and a measurable benchmark contract.

This is the final design baseline for the next implementation stage. It is not proof that Cognee, TurboVec, destructive-command hooks, Google ADK, NVIDIA agent runtime, LTX-2, Omma, vector-default retrieval, or provider-backed agents are installed or production-ready.

## What the current system already proved

FACT:

- WikiLLM is the canonical public-safe memory layer.
- LangGraph, LlamaIndex, CrewAI config/import, deterministic retrieval, safety checks, and provider-disabled workflow fixtures have existing narrow proof.
- Loop Engineering has state, retry caps, maker/checker separation, budgets, stop conditions, and handouts at L1.
- Graphify exists as generated structural reference.
- Obsidian/Nexus readiness is correctly separated into file/config, vault activation, live socket, tool schema, and actual tool-call states.
- The current four-vault Obsidian setup should not be replaced by turning the raw ArchFlow workspace into another vault.

GAP:

- Goal Engineering did not previously have a goal directory, lifecycle schema, verifier contract, or kill-switch state.
- A destructive-command defense-in-depth hook is not integrated.
- Vector-default retrieval, TurboVec, Cognee, provider-backed orchestration, continuous monitoring, and broad Nexus readiness are not proven.
- The existing public plan had stale dates and task/evidence-state drift.

## Target architecture

```text
Authority and safety envelope
  -> persistent bounded goal
  -> CAG core plus approved-corpus retrieval
  -> dependency graph and role/skill pack
  -> LangGraph state, checkpoints, and human gates
  -> Terra integration plus bounded Luna branches
  -> Loop Engineering maker/checker repair
  -> deterministic checks plus independent review
  -> reviewed WikiLLM promotion
  -> Graphify/Obsidian navigation refresh
  -> action-specific external gate
  -> benchmark and retrospective
```

LangGraph owns state. Loop Engineering repairs a bounded task. CrewAI may organize a named team inside a LangGraph node, but it must not become a second state controller. Goal Engineering persists the outcome without granting unlimited retries or broader authority.

## Knowledge and RAG conclusion

The final knowledge design is layered retrieval, not one giant database:

1. Routing and operating rules establish identity and safety.
2. WikiLLM holds reviewed durable conclusions, decisions, issues, runs, and insights.
3. Obsidian provides human semantic navigation, indexes, maps, and cross-links.
4. Graphify provides rebuildable structural relationships and blast-radius reference.
5. Nexus provides live vault operations only after schema and runtime proof.
6. LlamaIndex provides bounded hybrid retrieval with source IDs and lexical fallback.
7. Cognee may later provide operational graph/vector recall, but cannot replace WikiLLM.
8. TurboVec may later reduce vector index cost/size only if the same 20-query benchmark and metadata filters pass.

Do not ingest all available LLM or agent literature. Keep the existing curated survey collection as a discovery map, select 1-3 primary papers for a current decision, verify them, write a bounded synthesis, and promote only reusable conclusions. This is deeper operational knowledge than a large unreviewed paper dump.

## Obsidian method decision

Useful patterns from the reviewed Obsidian approaches are adopted selectively:

- index-first and hot/current context before deep reads;
- search-before-create;
- immutable raw sources and derived synthesis;
- provenance and bi-temporal history;
- reconciliation instead of silent overwrite;
- one write propagates to index, log, related note, and decision/run layers;
- advisory locking or single-writer ownership for multi-agent work;
- methodology chosen per corpus rather than one universal folder scheme.

Rejected or deferred:

- unattended agents using permission-bypass flags;
- automatic whole-vault mutation after every conversation or compaction;
- wholesale installation of a second vault methodology over the current contract;
- a broad REST MCP with delete authority when Nexus/filesystem already cover the need;
- whole-device or whole-Documents ingestion.

## Tool and ecosystem decisions

The Google/NVIDIA review is deliberately responsibility-based: build, deploy/scale, govern/safety, retrieval, and evaluate/optimize. It covers the official control-plane repositories most relevant to these roles, not every repository in either organization. The current Google Agent Starter Pack itself directs new projects to `agents-cli`, which is why both were not treated as equal active choices.

| Tool or method | Decision now | Why |
|---|---|---|
| Loop Engineering | Adopt at L1 | Already aligned with bounded attempts, state, maker/checker, and report-first maturity. |
| Goal Engineering | Adopt contract | Fills objective, done-condition, verifier, lifecycle, budget, and kill-switch gap. |
| `destructive_command_guard` | Trial later | Useful defense in depth; scanner did not finish, so no install or safe claim. Platform approval remains authoritative. |
| Google Agent Platform / ADK / agents-cli | Reference then trial | Strong build, scale, govern, optimize, evaluate, deploy, and observe patterns; avoid a second orchestrator without a measured use case. |
| NVIDIA skills | Adopt governance pattern | Prefer selected signed/evaluated skills and verify them; do not install the full catalog. |
| NeMo Agent Toolkit | Reference then trial | Best fit is future evaluation, profiling, observability, and optimization of a runnable agent. |
| NVIDIA RAG, Retriever, and Guardrails | Reference then role-specific trial | Useful for GPU-backed retrieval and programmable input/retrieval/execution/output safety, but heavier than the current bounded path. |
| Cognee | Defer to bounded trial | Promising graph/vector operational memory, but duplicates canonical memory unless carefully scoped. |
| TurboVec | Defer to benchmark | A compact vector index is useful only after stable embeddings, filters, persistence, and retrieval evidence. |
| LTX-2 | Defer to media role | Video/audio generation belongs to a resource- and license-gated content lane, not the control plane. |
| Omma | Defer to 3D prototype | Useful for website/scene prototyping after privacy, export, accessibility, and performance gates. |
| Legora | Product/UX reference | Its layered agent, knowledge, integration, interface, and governance story is relevant; it is not a dependency. |
| LaTeX/Typst | Trial as publishing skill | Use deterministic builds, citations, fonts, accessibility, and PDF checks for long-form content. |

## Performance comparison

No percentage gain is claimed in this review because the baseline and target have not run the same evaluation set.

Expected directional effects to test:

| Category | Unstructured baseline risk | Target architecture hypothesis |
|---|---|---|
| Agent execution | Ambiguous finish and repeated work. | Goal done conditions and state checkpoints reduce false completion and repetition. |
| Prompt/context | Large repeated prompts and conflicting rules. | CAG core, role-specific skills, and bounded retrieval reduce irrelevant context. |
| Token/cost | Frontier model used for every step. | Deterministic checks, local/bounded workers, caching, and escalated review reduce expensive calls. |
| Sequential reliability | Hidden dependency and state loss. | LangGraph checkpoints and proof gates make dependencies and resume state explicit. |
| Parallel reliability | Duplicate writers and merge conflicts. | Luna read-only branches plus one Terra integrator reduce write conflicts. |
| Memory quality | Appends, duplicates, stale claims, and shadow sources of truth. | Promotion, reconciliation, provenance, and superseded states improve auditability. |
| Retrieval | Unscoped semantic matches and missing sources. | Approved corpus, stable IDs, metadata filters, benchmark, and lexical fallback improve control. |
| Safety | Tool access implies authority. | Least privilege, platform approvals, tool adoption gates, and independent review separate capability from permission. |

The benchmark contract covers quality, reliability, tokens/context, retrieval, memory, parallel work, safety, cost/latency, recovery, and human effort. Improvement becomes a fact only after the paired baseline/candidate run.
