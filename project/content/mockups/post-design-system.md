# Static Post Design System

Status: planning design system
Date: 2026-07-10

## Purpose

Define public-safe static post and carousel visuals for ArchFlow content operations.

## Visual Direction

- Background: light gray or white operational surface.
- Panels: white with 8px radius or less.
- Text: dark ink, compact hierarchy, no oversized hero treatment for operational posts.
- Accent colors: red for risk/blocker, blue for process, green for pass, amber for review, muted gray for pending.
- Typography: Inter or system sans-serif.
- Mood: product-system and research-lab, not decorative marketing.

## Motifs

- Source-status chips.
- Redaction bars.
- Before/after artifact panels.
- FACT / INTERPRETATION / HYPOTHESIS / GAP labels.
- Stage rails.
- Proof cards.
- Review-gate stamps.
- Evidence receipt blocks.

## Prohibited Visuals

- Private screenshots.
- Raw transcripts.
- Account IDs, deployment IDs, private URLs, or personal identifiers.
- Customer logos or named target accounts without approval.
- Fake live dashboards or unsupported runtime claims.
- Decorative gradient blobs or visual noise that hides the artifact.

## Dimensions

| Format | Size | Notes |
|---|---|---|
| LinkedIn square | 1200 x 1200 | Single static educational post |
| LinkedIn portrait | 1080 x 1350 | High-density proof card |
| Carousel slide | 1080 x 1350 | 5 to 7 slides |
| X image | 1600 x 900 | One concept per image |
| Threads-style image | 1080 x 1350 | Reuse LinkedIn portrait safely |
| Short video cover | 1080 x 1920 | Static companion required |

## Accessibility

- Keep copy short enough to fit without overlap.
- Use strong contrast.
- Do not rely on color alone; include labels.
- Use stable margins and repeated grid positions.
