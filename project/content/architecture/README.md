# Content Agent Architecture

Status: planning specification; no social or API action authorized
Date: 2026-07-10

## Purpose

This architecture prepares evidence-led company content for ArchFlow's product knowledge-continuity system. It is not a generic AI-agency or PRD-generator content engine. Its content starts with a verified forcing moment, explains source/ownership/retrieval/governance work, and shows reviewable outputs such as evidence packs, onboarding maps, continuity handoffs, PRDs, task packets, and decision-memory updates.

## Operating Boundary

- The scope is planning, drafting, mockup specification, and review preparation only.
- No role posts, schedules, sends messages, calls an API, activates a provider, or writes to an external system.
- No role scrapes, browser-automates, downloads, or stores account, profile, follower, comment, direct-message, or engagement-graph data.
- Inputs must be public repo artifacts, official public sources, or public-safe owner-approved summaries. Outputs remain repo-relative public-safe summaries.
- A source or social observation supplies a topic or language hypothesis, never customer, revenue, ROI, demand, or runtime proof.

## Content Mix

The initial calendar counts ten publication candidates: seven static posts, two carousel/checklist posts, and one scripted short-video concept. The video's required static companion is one of the seven static posts. This is exactly the 70/20/10 mix; it is not a posting schedule.
